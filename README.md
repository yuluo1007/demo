# AgentScope homepage

A static, dependency-free site for [agentscope.io](https://agentscope.io). Three pages:

- **`index.html`** — the homepage
- **`community.html`** — the community page
- **`blog/`** — a tiny markdown-driven blog generated by `build_blog.py`

No build step, no node_modules, no framework. Shared CSS and JS live in `assets/`. The blog is the only thing that gets "built", and the builder is one Python file using only the standard library.

## Run locally

```bash
python3 -m http.server 8000
```

Then open http://localhost:8000.

**VS Code Live Server note:** the nav fragment is `assets/html/nav.inc` (not `.html`). Live Server only injects its livereload `<script>` into `.html`/`.htm` files — using `.inc` prevents that injection from corrupting the fetched fragment. If you switch to a different dev server, keep the `.inc` extension or the nav will break.

## Architecture

### Shared stylesheet — `assets/css/main.css`

All visual styles for the homepage and community page. Covers the nav, hero, project cards, sidebar, trusted-by marquee, community channels/roles, contributor grid, and blog listing/post pages.

### Nav fragment — `assets/html/nav.inc`

The nav link content is a plain HTML fragment loaded at runtime via `fetch()` by `nav-include.js` (homepage) or `nav-include-blog.js` (blog pages, which sit one directory deeper). The fragment contains four project group dropdowns (Building, Intelligence, Agents & Playgrounds, Dev Tools) plus Blogs and Community links.

To update the nav dropdowns, edit `assets/html/nav.inc` directly — no rebuild needed, just refresh.

### Left sidebar — `assets/js/content-nav.js`

A fixed sidebar sits at the left of the viewport throughout the page. It has four buttons, one per project group. Each button:

- **Hover** — pops out a dropdown menu to the right with the same project list as the corresponding nav dropdown.
- **Click** — smooth-scrolls to that group's header in the main frame.
- **Scroll tracking** — the active button updates as the user scrolls past each group's `id` anchor.

The sidebar is hidden below 1280 px (hamburger breakpoint).

## The homepage (`index.html`)

Sections in order:

1. **Top nav** — brand logo, Blogs + Community links (right-aligned), social icons (GitHub, X, Discord, RedNote, DingTalk). Project dropdowns are loaded from `nav.inc`.
2. **Video hero** — background clips from `videos/` cycle behind the title and tagline.
3. **Left sidebar** — fixed navigation panel; see above.
4. **Projects section** — four groups of project cards (2 per row), each group with a heading and description:
   - *Agent Frameworks and Runtimes* — agentscope, agentscope-java, HiClaw
   - *Intelligence and Evolving* — ReMe, OpenJudge, Trinity-RFT, TuFT
   - *Agents and their Playgrounds* — QwenPaw, DojoZero, PawFriends
   - *Developer Experience* — AgentScope Spark Design
5. **Trusted by** — partner logo marquee.

### Common edits

**Add or update a nav dropdown item.** Edit `assets/html/nav.inc`. Each dropdown is a `<div class="nav-projects">` block containing a `<div class="nav-projects-grid">` with `<a class="nav-project-item">` entries. Also update the matching sidebar dropdown in `index.html` (same content, inside the `<div class="group-nav-btn">` blocks).

**Add a project card.** Find the right `projects-group-header` in `index.html` and insert a new card after it:

```html
<div class="project-card">
    <a class="project-image" href="<primary-link>" target="_blank" rel="noopener">
        <img src="<preview-image>" alt="..." loading="lazy" />
    </a>
    <div class="project-body">
        <div class="project-header">
            <h3>Name</h3>
            <span class="project-tag">Category</span>
        </div>
        <p>Short description.</p>
        <div class="project-links">
            <a href="<github-url>" target="_blank" rel="noopener">
                …<span class="label">github.com/…</span><span class="arrow">→</span>
            </a>
        </div>
    </div>
</div>
```

Add `logo` to `.project-image` (`<a class="project-image logo">`) when the preview is a logo rather than a screenshot — switches `object-fit` to `contain` and adds a gradient background.

**Swap a hero video.** Drop the `.mp4` into `videos/` and edit the `videos = […]` array in `assets/js/video.js`.

**Adjust hero darkness.** Edit the `rgba` alpha values on `.hero-overlay` in `assets/css/main.css`.

**Swap a partner logo.** Change the `<img>` URLs inside `<div class="trusted-marquee">` in `index.html`. Both `<div class="trusted-track">` blocks (original + duplicate for the seamless loop) need updating.

**Update contributors.** Edit `data/contributors.json` directly, or rerun the fetcher:

```bash
python3 scripts/build_contributors.py
```

## The blog

Posts are markdown files in `blog/posts/` with YAML-style frontmatter. The builder generates one HTML page per post plus an index page listing them newest-first.

### Add a post

1. Create `blog/posts/<slug>.md`. The slug becomes the URL: `<slug>.md` → `/blog/<slug>.html`.
2. Add frontmatter at the top:

   ```markdown
   ---
   title: My Post Title
   date: 2026-04-28
   excerpt: One-line summary shown on the blog index.
   author: Optional Author
   hero: images/my-hero.png
   category: Engineering
   ---

   Body in markdown…
   ```

   Required: `title`, `date` (`YYYY-MM-DD`). Optional: `excerpt`, `author`, `hero` (path relative to project root), `category`. Wrap values containing colons in double quotes.

3. Run the builder:

   ```bash
   python3 build_blog.py
   ```

   Writes `blog/<slug>.html`, rewrites `blog/index.html`, and deletes stale HTML for removed posts.

### Supported markdown

The builder is a small custom parser in `build_blog.py` (no external libraries). It handles:

| Syntax | Output |
|--------|--------|
| `# Heading` through `###### Heading` | `<h1>`–`<h6>` |
| Paragraphs | `<p>` |
| `**bold**`, `*italic*` | `<strong>`, `<em>` |
| `` `inline code` `` | `<code>` |
| ` ```lang `…` ``` ` | `<pre><code class="lang-…">` |
| `[text](url)` | `<a rel="noopener">` |
| `![alt](src)` | `<img loading="lazy">` |
| `- item` / `* item` / `+ item` | `<ul><li>` |
| `1. item` | `<ol><li>` |
| `> quote` | `<blockquote>` |
| `---` / `***` (alone on a line) | `<hr>` |
| GFM tables (`\| h \| h \|` + `\|---\|---\|`) | `<table>` |

Not supported: nested lists, footnotes, task lists, autolinks, raw HTML in bodies. Swap `md_to_html()` in `build_blog.py` for `mistune` or `python-markdown` if needed.

### Edit the look

- **Blog listing and post typography** → `blog/blog.css`
- **Page chrome (nav, footer, wrapper)** → `PAGE_OPEN` / `PAGE_CLOSE` in `build_blog.py`
- **Card / article structure** → `render_post_page()` / `render_index_page()` in `build_blog.py`

After any change to `build_blog.py` or `blog/blog.css`, rerun `python3 build_blog.py`.

## Deployment

Everything is static — point any static host (S3, Cloudflare Pages, GitHub Pages, Netlify, nginx) at the project root. Ensure the host serves `blog/index.html` for `/blog/` requests (most do by default).

The `videos/` directory is heavy; consider serving those files from a CDN and updating the `videos = […]` array in `assets/js/video.js` to point at the CDN URLs.

No SSR, no API, nothing to configure. The entire artifact is `index.html`, `community.html`, `assets/`, `blog/`, `images/`, `videos/`, `data/`, and `logo.png`.

### GitHub Pages (this repo)

Push to **GitHub** (`https://github.com/agentscope-ai/agentscope-homepage-experiment`). In the repo go to **Settings → Pages**, set **Build and deployment → Source** to **GitHub Actions**. On each push to `main` or `master`, the workflow [.github/workflows/deploy-pages.yml](.github/workflows/deploy-pages.yml) runs `python3 build_blog.py` and publishes the site.

The default URL for a project Pages site is `https://agentscope-ai.github.io/agentscope-homepage-experiment/`. Nav links use paths relative to that root so they work under this prefix without extra configuration.
